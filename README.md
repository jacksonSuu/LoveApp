## 功能规则

- 每位用户注册后可在右上角「契约」弹窗中发起契约申请（按昵称）
- 被申请人可在契约弹窗中接收申请并选择接受/拒绝
- 未绑定契约：每天发放 50 次测试抽奖，奖品仅展示不会进入背包
- 已绑定契约：解锁契约任务，并启用真实抽奖（奖品会进入背包）
- 任务赠送次数：每天最多累计 10 次
- 当前任务仅开放：签到任务、抽奖任务；未绑定时契约任务显示锁定中
- 抽奖方式：单抽 / 五连抽
- 引导页：新用户首次进入需先注册昵称（全屏不可关闭）
- 昵称规则：支持任意字符（含符号），但不能是 `root`，且必须唯一
- 自动进入：首页是否展示引导页以本地 `user token` 为判断条件

## 主界面布局

- 左上角：标题
- 右上角：剩余次数与任务赠送进度
- 右上角：管理员入口图标（🛠️）
- 主区域：抽奖动画与抽奖按钮
- 任务入口：主界面图标点击后弹出任务套餐
- 抽奖结果：弹窗展示

## 管理员面板

- 入口：点击右上角 `🛠️`
- 登录校验：必须输入账号密码才能修改配置
- 默认账号：`root`
- 默认密码：`root`
- 支持操作：
	- 修改基础次数、任务奖励上限、特殊任务解锁门槛
	- 修改奖池 / 日常任务 / 特殊任务（JSON）
	- 修改管理员密码
	- 退出管理员登录

- 会话有效期：登录后默认 2 小时，过期后需要重新登录
- 安全交互：保存配置前会弹出二次确认

> 保存配置后会立即生效，并重置当天进度。

## 启动方式

1. 安装依赖：`npm install`
2. 开发运行：`npm run start:dev`
3. 浏览器访问：`http://localhost:3000`

## 目录说明

- `src/lottery/*`：抽奖与任务后端逻辑
- `public/*`：前端页面

## 状态存储

- 当前版本已切换为 MySQL 存储
- 按用户维度保存抽奖状态（通过请求头 `x-user-id` 区分）
- 每日会自动按日期重置该用户的任务与次数

## MySQL 配置

在 `next-app/.env` 中配置：

- `MYSQL_URL`（推荐）
- 或 `MYSQL_HOST` / `MYSQL_PORT` / `MYSQL_USER` / `MYSQL_PASSWORD` / `MYSQL_DATABASE`

## 可配置项

可通过管理员面板在线修改，后端对应字段在 `src/lottery/lottery.service.ts` 的 `config` 中：

- `baseDailyChances`
- `maxTaskBonus`
- `specialUnlockTarget`
- `prizePool`
- `dailyTasks` / `specialTasks`
