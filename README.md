# 恋爱抽奖站（Nest 全栈版）

单项目同时开发前后端：

- 后端：Nest API（任务、抽奖、状态管理）
- 前端：`public/` 静态页面（由 Nest 统一托管）

## 功能规则

- 每天默认抽奖次数：50 次
- 任务赠送次数：每天最多累计 10 次
- 任务类型：日常任务 + 特殊任务（任务以“套餐”形式在弹窗展示）
- 解锁条件：累计完成 6 次日常任务后解锁特殊任务
- 抽奖方式：单抽 / 五连抽

## 主界面布局

- 左上角：标题
- 右上角：剩余次数与任务赠送进度
- 右上角：管理员入口图标（🛠️）
- 主区域：抽奖动画与抽奖按钮
- 任务入口：主界面图标点击后弹出任务套餐
- 抽奖结果：弹窗展示

## 管理员面板

- 入口：点击右上角 `🛠️`
- 登录校验：必须输入账号密码才能修改配置
- 默认账号：`root`
- 默认密码：`root`
- 支持操作：
	- 修改基础次数、任务奖励上限、特殊任务解锁门槛
	- 修改奖池 / 日常任务 / 特殊任务（JSON）
	- 修改管理员密码
	- 退出管理员登录

- 会话有效期：登录后默认 2 小时，过期后需要重新登录
- 安全交互：保存配置前会弹出二次确认

> 保存配置后会立即生效，并重置当天进度。

## 启动方式

1. 安装依赖：`npm install`
2. 开发运行：`npm run start:dev`
3. 浏览器访问：`http://localhost:3000`

## 目录说明

- `src/lottery/*`：抽奖与任务后端逻辑
- `public/*`：前端页面

## 状态存储

- 当前版本使用服务端内存状态（服务重启会重置）
- 每日会自动按日期重置任务与次数

## 可配置项

可通过管理员面板在线修改，后端对应字段在 `src/lottery/lottery.service.ts` 的 `config` 中：

- `baseDailyChances`
- `maxTaskBonus`
- `specialUnlockTarget`
- `prizePool`
- `dailyTasks` / `specialTasks`
